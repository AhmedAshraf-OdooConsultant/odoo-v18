# AKSHAB - POS Sales Report | تقرير مبيعات نقاط البيع التفصيلي

أدون مخصص لشركة **أخشاب البخور للتجارة** — أودو 18 إنتربرايز (Odoo.sh).

## المحتوى
تقرير PDF عربي (RTL) بالهوية البصرية للشركة (أخضر داكن `#1B4332` وذهبي `#C6A46C`) يُطبع من نافذة تحديد **الفترة** و**نقاط البيع**، ويشمل:

1. **إجمالي المبيعات حسب نقطة البيع** — الإجمالي، عدد الفواتير، عدد العملاء (جدد / قدامى) + صف إجمالي.
2. **فواتير المبيعات** — التاريخ، رقم الفاتورة، نقطة البيع، العميل، الهاتف، الإجمالي (تنازلياً).
3. **فواتير مرتجعات المبيعات** — نفس الأعمدة (تنازلياً حسب القيمة).
4. **مبيعات المنتجات حسب الفئة** — الكمية والإجمالي، مرتبة تنازلياً حسب الكمية.
5. **مبيعات الموظفين حسب نقطة البيع** — نفس مؤشرات القسم الأول لكل موظف + صف إجمالي.
6. **تحليل الخصومات حسب نقطة البيع** — برامج الخصم بالاسم + «خصومات عامة» (الخصم اليدوي للكاشير) + الإجماليات.

## التثبيت على Odoo.sh
1. انسخ مجلد `akshab_pos_sales_report` إلى مستودع Git الخاص بالمشروع (بجانب باقي الأدونات المخصصة).
2. اعمل commit ثم push إلى فرع staging للاختبار.
3. من قائمة **التطبيقات** ← حدّث قائمة التطبيقات ← ثبّت **AKSHAB - POS Sales Report**.
4. الوصول للتقرير: **نقاط البيع ← التقارير ← تقرير المبيعات التفصيلي**.

## الاعتماديات
`point_of_sale`, `pos_loyalty`, `account` — كلها قياسية ومثبتة تلقائياً.

## ملاحظات فنية (منطق البيانات)
- **مصدر البيانات:** طلبات نقاط البيع `pos.order` بالحالات `paid / done / invoiced` ضمن الفترة ونقاط البيع المحددة. المرتجعات = الطلبات ذات الإجمالي السالب.
- **رقم الفاتورة:** رقم القيد المحاسبي `account.move` إن كان الطلب مفوتراً، وإلا مرجع طلب نقطة البيع.
- **العملاء الجدد / القدامى:** نفس منطق حقل الاستوديو `x_studio_selection_field_57p_1j3eak00d` (New Customer ?) ولكن على مستوى فترة التقرير: العميل يُعتبر **جديداً** إذا لم توجد له أي فاتورة عميل مرحّلة (`out_invoice / posted`) بتاريخ سابق لبداية الفترة. الطلبات بدون عميل (بيع نقدي) لا تدخل في عدّ العملاء.
- **الموظف:** حقل `employee_id` (الكاشير) إذا كانت وحدة `pos_hr` مفعّلة، وإلا مستخدم الطلب.
- **فئة المنتج:** التصنيف الداخلي للمنتج `product.categ_id`. (يمكن تحويله بسهولة إلى فئة نقطة البيع `pos_categ_ids` عند الطلب.)
- **الخصومات:**
  - بنود المكافآت السالبة من برامج الولاء/العروض (`reward_id`) تُجمع باسم البرنامج.
  - نسبة الخصم اليدوية على البند (`discount %`) تُجمع تحت اسم **«خصومات عامة»** وتُحسب: `سعر الوحدة × الكمية × النسبة ÷ 100` (قبل الضريبة).

## المرحلة التالية (مقترح)
الإرسال التلقائي اليومي/الأسبوعي عبر واتساب: إضافة `ir.cron` يُنشئ الـ wizard بفترة (اليوم/الأسبوع المنصرم)، يُصيّر الـ PDF عبر `ir.actions.report._render_qweb_pdf`، ثم يرسله عبر وحدة WhatsApp في أودو 18 إنتربرايز (`whatsapp` module) بقالب موافق عليه في Meta.
